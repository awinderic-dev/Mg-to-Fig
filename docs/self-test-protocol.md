# 自我测试规程（Mg-to-Fig）

## 1. 目的与适用范围

- 目的：在 MasterGo -> Figma 迁移迭代中，稳定验证“可编辑性、还原度、可追溯性”。
- 适用范围：MVP、组件增强、变量增强三个阶段的开发与回归。

## 2. 角色分工

- 开发者：执行 L1/L2/L3，输出测试记录与问题清单。
- 设计师或需求方：参与 L4 关键页面终验，确认可接受降级项。

## 3. 测试分层与退出条件

## 3.1 L1 冒烟测试（每次提交都做）

- 用例：最小样例（1 页，含文本+矩形+图片）。
- 通过条件：
  - 导出 JSON 成功。
  - 导入 Figma 成功。
  - 页面可编辑，且插件不崩溃。
  - 诊断报告可生成。

## 3.2 L2 功能测试（每日或功能合并前）

- 用例覆盖：
  - 节点：Frame/Group/Text/Rectangle/Vector/Image。
  - 样式：fill/stroke/radius/effect/opacity/text style。
  - 布局：基础约束与 Auto Layout 映射。
  - 组件：主组件/实例基础恢复，失败项有降级说明。
  - 资源：小图内嵌、大图外链/分包加载。
- 通过条件：核心用例通过率 >= 95%，且无 P0 阻断问题。

## 3.3 L3 回归测试（每次版本候选必做）

- 用例：固定回归样例集（简单/复杂/组件实例）。
- 通过条件：
  - 与上一版本相比无明显退化。
  - 失败项数量不上升或有明确可接受说明。
  - 已知问题仍可定位到 nodeId。

## 3.4 L4 发布前测试（与设计师联合）

- 用例：黄金页面与高价值业务页面。
- 通过条件：
  - 视觉验收达标（见 PRD 10.1）。
  - 语义验收达标（见 PRD 10.2）。
  - 允许降级项经过确认并记录。

## 4. MCP 辅助验证建议（可选）

- 可选借助双方页面读取能力进行“结果复核”：
  - 读取 MasterGo 源页面关键信息（层级、名称、核心样式）。
  - 读取 Figma 导入后页面结构与节点属性。
  - 对比输出：节点数、层级路径、关键样式字段、组件映射情况。
- 目标：减少仅靠肉眼检查的遗漏，提高复测效率。

## 5. 需要你配合的检查点

- 提供并冻结样例文件（至少 3 份）和黄金页面。
- 确认字体替换表（例如 PingFang -> Inter 或系统字体）。
- 在 L4 阶段做终验并确认验收口径：
  - 位置/尺寸、圆角、阴影、描边、透明度等参数不允许变化。
  - 文本换行差异可接受。
  - 字号、字重、行高参数不允许变化（仅字体映射导致的换行变化可接受）。

## 6. 缺陷分级与处理时限

- P0：导入失败、崩溃、页面不可编辑。需立即修复。
- P1：主要布局错位、样式大面积异常、组件关系明显错误。需版本内修复。
- P2：局部视觉偏差、可手工修复项。可排期修复。
- P3：文案、命名、提示语优化。按优先级排队。

## 8. 发布门禁（建议）

- L1/L2/L3 必须全部通过。
- L4 完成并得到需求方确认。
- 未关闭的 P0/P1 数量为 0。
