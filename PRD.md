# PRD v1：MasterGo → Figma 迁移工具（Mg-to-Fig）

## 1. 文档信息

- **版本**：v1.0（草案）
- **定位**：先完成可用 MVP，再逐步补强组件与变量语义
- **已确认策略**：
  - 优先级：`MVP 优先`
  - 验收模式：`视觉 + 语义双维度平衡`
  - 交付形态：`MasterGo 插件 + Figma 插件`
  - 资源策略：`混合（小图内嵌，大图外链/分包）`

## 2. 背景与问题

当前缺少稳定的 MasterGo 到 Figma 的高保真迁移路径，导致团队难以在 Figma 持续编辑历史资产。  
目标是在可控有损前提下，建立一套可复用迁移工具链，尽量保留：

- 样式与布局结构
- 组件层级与实例关系
- 变量/样式引用的可追溯信息（逐步增强到半自动绑定）

## 3. 目标与成功标准

### 3.1 业务目标

- 降低跨工具迁移成本与人工重建成本
- 保证迁移后在 Figma 可继续编辑与协作
- 形成可迭代的工程化迁移能力

### 3.2 产品目标（MVP）

- 支持页面/画板/选中树导出导入
- 支持常见图层与样式还原（文本、矩形、基础矢量、图片）
- 迁移结果在 Figma 中可编辑，不是纯图片快照

### 3.3 成功指标（建议）

- **视觉一致度**：关键页面元素位置/尺寸/样式还原度 ≥ 90%
- **语义一致度**：可识别并恢复的组件实例关系覆盖率 ≥ 70%（MVP 可先记录可追溯信息）
- **迁移可用性**：单页面迁移成功率 ≥ 95%
- **效率**：同等页面人工重建工时下降 ≥ 50%

## 4. 范围定义

### 4.1 In Scope（v1）

- MasterGo 插件：导出页面/选中节点为中立 JSON
- Figma 插件：读取 JSON 并重建可编辑节点
- 核心样式：填充、描边、圆角、阴影、透明度、基础文本样式
- 布局：Frame/Group 层级、基础约束/Auto Layout 映射
- 组件：主组件/实例的基础重建（单层优先）
- 变量：保留 token 原始路径/名称与最终值（可追溯）

### 4.2 Out of Scope（v1）

- 评论、权限、历史版本、复杂原型交互
- 100% 无损语义映射承诺
- 大型跨文件企业级组件库完整自动迁移

## 5. 目标用户与核心场景

- **目标用户**：产品设计师、设计工程师、设计系统维护者
- **核心场景**：
  1. 设计师选中 MasterGo 页面导出JSON
  2. 在 Figma 插件中导入JSON
  3. 得到可编辑页面，并得到报告告知成果成功/失败数量，然后手动修复少量差异
  4. 对变量进行二次绑定（后续增强）

## 6. 功能需求（FR）

- **FR-01 导出入口**：MasterGo 插件支持“导出当前选择 / 当前页面”
- **FR-02 中立 Schema**：统一 `schemaVersion`，包含节点、样式、布局、资源、组件、变量引用、错误信息
- **FR-03 导入重建**：Figma 插件按树结构创建节点并保留命名层级
- **FR-04 样式映射**：将 MasterGo 常见样式映射到 Figma 属性
- **FR-05 组件映射**：支持主组件与实例基础关系恢复；无法恢复时落地为可编辑节点并标记
- **FR-06 变量可追溯**：保存 `tokenPath/tokenName/resolvedValue`，用于后续绑定
- **FR-07 诊断与日志**：输出按 `nodeId` 定位的问题列表（丢失字段、降级原因、资源异常）
- **FR-08 资源混合策略**：小图 base64 内嵌；大图外链/分包并可重试加载

## 7. 非功能需求（NFR）

- **性能**：中型页面（~500 节点）导入耗时可控（目标 < 20s）
- **稳定性**：失败不应导致 Figma 文档不可编辑；支持部分成功导入
- **可维护性**：Schema 可版本演进，向后兼容至少一个小版本
- **可观测性**：关键步骤有结构化日志（导出、解析、导入、降级）

## 8. 数据契约（JSON v0 建议）

- `schemaVersion`
- `documentMeta`（来源、时间、页面标识）
- `nodes[]`（id/type/name/children/layout/style/text/vector/image/componentRef）
- `assets[]`（id,type,size,transport:inline|external,uri/data）
- `tokens[]`（tokenId,path,name,resolvedValue,usageNodeIds）
- `diagnostics[]`（level,code,nodeId,message,fallbackApplied）

## 9. 迁移流程（用户视角）

1. 在 MasterGo 选择迁移范围并导出
2. 插件生成 JSON + 资源包（混合策略）
3. 在 Figma 运行导入插件并选择 JSON
4. 插件重建页面并输出诊断报告
5. 设计师核验关键页面并修复残差
6. （后续）执行变量绑定器完成语义增强

## 10. 验收标准（双维度）

### 10.1 视觉验收

- 关键页面中，主框架层级与布局关系一致
- 颜色/圆角/阴影/描边/透明度等视觉参数需与源文件保持一致，不接受数值变化
- 文本换行差异可接受（需字体映射表），但字号、字重、行高参数不允许被自动改写

### 10.2 语义验收

- 组件命名与层级保留可读性
- 可恢复实例建立实例关系；不可恢复项有明确降级标记
- 变量引用至少可追溯（路径/名称可查）

### 10.3 失败处理

- 任一节点失败不应中断全量导入
- 失败项必须在诊断中可定位到 `nodeId`

## 11. 里程碑

- **M1（MVP）**：页面/样式/基础节点可迁移可编辑
- **M1.1（稳定性增强）**：资源资产化、字体预检、分片导入会话恢复
- **M2（组件增强）**：主组件/实例关系恢复能力提升
- **M3（变量增强）**：变量/样式半自动绑定能力

## 12. 风险与应对

- **字体差异**：建立字体替换表 + 文本容差规则
- **布局模型差异**：无法 1:1 时降级为 Auto Layout 并打标
- **资源体积/超时**：混合资源策略 + 分包 + 重试
- **API 变动**：Schema 版本化 + 映射层隔离

## 13. 自我测试规程（开发与回归）

- 目标：每次迭代都能快速判断“是否可用、差异在哪、是否可发布”。
- 执行方式：开发者自测为主，设计师/你配合做关键页面终验。
- 详细步骤与模板见：`docs/self-test-protocol.md`。

### 13.1 自测分层

- **L1 冒烟测试**：单页面最小样例可导入、可编辑、无阻断报错。
- **L2 功能测试**：节点、样式、布局、组件、资源链路按用例通过。
- **L3 回归测试**：历史样例跑全量，比较本次与上次结果是否退化。
- **L4 发布前测试**：关键页面人工验收，确认视觉+语义门槛达标。

### 13.2 你需要配合的环节

- 提供 3 份样例文件及黄金页面（简单/复杂/含组件实例）。
- 对关键页面做人工终验（字体、文本换行、组件语义）。
- 对“允许降级项”签字确认（避免后续口径变化）。

## 14. 输入输出规格（实施口径）

### 14.1 输入规格（MasterGo 导出侧）

- 输入单位：`当前页面` 或 `当前选中节点树`
- 最低输入要求：
  - 节点具备稳定 `id`、`type`、`name`
  - 样式字段可读（fill/stroke/effect/opacity/text）
  - 布局字段可读（父子关系、位置尺寸、约束/自动布局信息）
- 资源输入：
  - 小图以内嵌数据传输
  - 大图以外链或分包资源索引传输

### 14.2 输出规格（Figma 导入侧）

- 输出对象：可编辑页面结构（非位图快照）
- 输出结果包含：
  - 导入后的页面/节点树
  - 诊断报告（成功/失败数量、失败 nodeId、降级原因）
  - token 可追溯信息（路径、名称、解析值）

## 15. 字体映射规则（实现基线）

- 映射优先级：
  1. 同名精确匹配（family + style）
  2. 同 family、最近 style/weight 匹配
  3. 名称相近匹配（同义或缩写）
  4. 系统回退字体
- 约束：
  - 允许文本换行差异
  - 不允许自动改写字号、字重、行高参数
  - 映射过程需记录命中策略，写入诊断日志

## 16. 错误码与诊断（v1）

- `E_NODE_UNSUPPORTED`：节点类型暂不支持，按降级策略处理
- `E_STYLE_UNMAPPABLE`：样式字段无法等价映射，保留原值并告警
- `E_LAYOUT_CONFLICT`：布局模型冲突，降级为可编辑近似结构
- `E_COMPONENT_BIND_FAIL`：组件/实例绑定失败，降级为普通节点
- `E_ASSET_MISSING`：资源缺失或加载失败
- `E_FONT_FALLBACK`：字体未命中，触发回退映射
- `E_TOKEN_BIND_PENDING`：变量绑定未执行，仅保留可追溯信息

## 17. 最终方案确认清单（立项前）

- 交付形态：MasterGo 插件 + Figma 插件
- 迁移策略：双端插件 + 中立 JSON（版本化）
- 资源策略：小图内嵌 + 大图外链/分包
- 验收口径：参数不可变（颜色/圆角/阴影/描边/透明度/字号/字重/行高）；仅允许换行差异
- 质量门禁：L1/L2/L3 全通过 + L4 人工终验通过 + P0/P1 为 0
